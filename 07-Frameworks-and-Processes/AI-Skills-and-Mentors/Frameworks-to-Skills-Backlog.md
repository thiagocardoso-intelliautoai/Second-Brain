# Frameworks to Skills Backlog

## Decisao da Wave 4

A pergunta original da Wave 4 era:

> Quando voce usa esses frameworks na pratica, qual tipo de saida voce mais quer gerar: diagnostico para cliente, plano tecnico, post, proposta, aula, ou documento interno?

A decisao operacional e que essa pergunta ainda esta vaga demais para justificar transformar todos os frameworks em templates, prompts ou skills agora.

Sem uso recorrente claro, "toolificar" os frameworks criaria manutencao falsa: mais estrutura, mais campos, mais exemplos artificiais e pouco ganho real.

Por enquanto, os frameworks desta pasta devem funcionar como:

- lentes de pensamento;
- checklists leves;
- insumos para projetos, clientes e posts;
- materia-prima para skills futuras.

Eles so devem virar ferramenta operacional quando existir repeticao real.

## Regra de promocao

Um framework pode virar prompt, template ou skill quando tiver:

- caso de uso concreto;
- input recorrente;
- output esperado observavel;
- criterio de qualidade;
- 2 ou 3 usos reais, ou uma demanda viva que claramente vai se repetir;
- projeto, cliente ou frente onde a skill sera testada.

Evitar promocao quando:

- o framework ainda e so tese;
- o tipo de output esta indefinido;
- existe apenas um exemplo especulativo;
- a skill duplicaria um processo ja existente;
- o custo de manutencao seria maior que o ganho.

## Como usar agora

- Antes de diagnostico, automacao ou documentacao tecnica, ler o framework relevante como lente.
- Usar as secoes existentes como checklist, nao como formulario obrigatorio.
- Salvar outputs reais dentro do projeto, cliente ou pasta de contexto onde eles foram usados.
- Quando o mesmo padrao aparecer mais de uma vez, criar uma skill local primeiro.
- Promover para skill global somente depois de usos reais em mais de um contexto ou quando ficar claro que o metodo e transversal.

## Melhor pergunta para uma IA fazer depois

Em vez de perguntar genericamente qual output Thiago quer, perguntar:

```text
Qual uso real se repetiu?
Me mostra 1 a 3 outputs concretos que voce ja gerou ou tentou gerar com esse framework.
A partir disso, eu decido se vira checklist, template, skill local ou skill global.
```
