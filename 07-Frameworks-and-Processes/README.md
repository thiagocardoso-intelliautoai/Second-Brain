## Regra central

Framework bom precisa orientar decisao e execucao. Se e so nome bonito, nao entra aqui.

## Como decidir onde colocar

- Se o processo serve principalmente a um dominio canonico, use a subpasta com o mesmo nome do dominio.
- Se o processo nasceu de cliente ou mentoria, coloque em `05-Clients-and-Mentoring` somente uma versao anonima e reutilizavel.
- Se o processo e uma skill, mentor, politica de skill ou backlog de promocao, use `AI-Skills-and-Mentors`.
- Se o processo organiza o proprio Second Brain, use `Second-Brain-System`.
- Se servir a mais de um dominio, salve no uso principal atual e crie links cruzados nos READMEs.

## Politica de promocao para skills

Nem todo framework precisa virar ferramenta operacional imediatamente.

Por enquanto, frameworks podem funcionar como lentes, checklists leves e materia-prima para skills futuras. A transformacao em prompt, template ou skill so deve acontecer quando houver uso recorrente claro, input definido, output esperado e criterio de qualidade observavel.
