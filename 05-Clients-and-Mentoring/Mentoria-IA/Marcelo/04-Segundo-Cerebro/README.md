# Marcelo - Fase 4 - Segundo Cerebro

Status: decisao de abordagem registrada em 2026-05-21.

Use esta fase quando a mentoria trabalhar base de contexto, organizacao por cliente/processo, reaproveitamento de decisoes e uso de IA com memoria operacional.

## Decisao de abordagem

Para o caso do Marcelo, o "segundo cerebro" nao deve comecar como uma estrutura complexa em VS Code, repositorio local, pastas Markdown avancadas ou sistema proprio de manutencao.

A implementacao inicial deve ser simples e operacional:

- um Claude Project como base de contexto;
- arquivos ou notas essenciais sobre processo, cliente, proposta, decisoes e proximos passos;
- uma ou mais Skills para executar tarefas recorrentes, comecando pela skill de primeira reuniao com cliente -> ata/diagnostico -> base de proposta;
- instrucoes simples de manutencao da base;
- criterios claros sobre o que pode ou nao entrar na IA.

O objetivo e ensinar o principio do segundo cerebro, nao a arquitetura completa usada por Thiago.

## Posicionamento para Marcelo

Explicar para Marcelo que:

```text
Segundo cerebro aqui nao e uma ferramenta sofisticada. E uma base de contexto minima para a IA entender melhor seus clientes, processos, decisoes e materiais reutilizaveis. Para comecar, vamos fazer isso com Claude Project + Skills.
```

Essa abordagem reduz friccao, evita overengineering e conecta diretamente com o valor validado na mentoria: transformar reunioes reais em ativos reutilizaveis para diagnostico, proposta e acompanhamento.

## O que evitar nesta fase

- Nao apresentar VS Code como caminho inicial.
- Nao ensinar estrutura local complexa antes de haver uso real.
- Nao transformar a fase em aula de organizacao de arquivos.
- Nao criar uma arquitetura maior do que os ativos que Marcelo ja usa.
- Nao vender "segundo cerebro" como sistema completo antes de provar valor com Project + Skills.

## Quando evoluir para algo mais robusto

Considerar uma estrutura mais avancada apenas se Marcelo:

- testar a skill de proposta em casos reais;
- acumular ativos que precisa reaproveitar;
- sentir dor clara de organizacao entre varios clientes/processos;
- precisar de mais controle, portabilidade ou padronizacao do que o Claude Project oferece.

Enquanto isso nao acontecer, manter a fase como base de contexto simples, util e facil de adotar.
