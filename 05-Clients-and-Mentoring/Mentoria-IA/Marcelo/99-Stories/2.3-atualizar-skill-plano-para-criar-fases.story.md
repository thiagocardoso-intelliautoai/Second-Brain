# Story 2.3 - Atualizar skill de plano para criar fases

Status: Draft

## Story

Como Thiago, quero que a skill `criar-plano-de-mentoria-a-partir-do-diagnostico` crie automaticamente a estrutura de pastas a partir das fases validadas do plano da mentoria, para que cada novo mentorado fique organizado de acordo com a propria jornada, sem engessar todos no mesmo modelo.

## Contexto

No caso Marcelo, a pasta foi reorganizada em cinco fases:

```text
00-Diagnostico
01-Fundamentos
02-Automacao-Assistida
03-Workflows-Agents
04-Segundo-Cerebro
```

Essa estrutura e um exemplo do caso Marcelo, nao um padrao obrigatorio para todos os mentorados. Normalmente a mentoria pode seguir algo parecido, mas cada mentorado pode ter fases diferentes dependendo do diagnostico, da maturidade, do objetivo e do plano validado.

A estrutura nao deve ser criada antes de Thiago validar as fases do plano. A ordem correta e:

1. a skill cria ou revisa o plano;
2. Thiago valida a jornada/fases;
3. a skill extrai as fases validadas do plano;
4. a skill cria as pastas correspondentes na pasta do mentorado;
5. a skill move ou orienta a organizacao dos arquivos existentes sem apagar contexto.

## Skill afetada

- `C:/Users/thiag/.codex/skills/criar-plano-de-mentoria-a-partir-do-diagnostico/SKILL.md`

## Acceptance Criteria

1. A skill passa a ter uma etapa explicita de `Organizar fases depois da validacao`.
2. A skill so cria pastas apos confirmacao/validacao das fases por Thiago.
3. A skill usa nomes de pastas derivados das fases realmente validadas no plano, com prefixo numerico ordenado.
4. A skill trata a estrutura do Marcelo apenas como exemplo ou fallback quando o plano validado tiver fases equivalentes.
5. A skill nunca assume que todo mentorado tera as mesmas cinco fases do Marcelo.
6. A skill pede validacao/pergunta para Thiago se as fases do plano estiverem ambiguas, sobrepostas ou sem ordem clara.
7. A skill nao move transcricoes brutas sem preservar proveniencia.
8. A skill nao sobrescreve pastas existentes.
9. A skill atualiza ou sugere atualizar o README do mentorado com os links das fases.
10. A skill e validada com `quick_validate.py`.
