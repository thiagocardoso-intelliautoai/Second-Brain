# Marcelo - Stories da Mentoria

Stories para estruturar melhorias, criacao de skills e evolucao do metodo de mentoria a partir do caso Marcelo.

## Regra de uso

- Criar stories pequenas, executaveis e com criterios claros.
- Separar o que depende de validacao do Thiago.
- Nao transformar pesquisa futura em decisao sem checkpoint humano.
- Quando uma story gerar metodo reutilizavel, promover uma versao anonima para `05-Clients-and-Mentoring/Mentoria-IA/05-Skills/` ou `07-Frameworks-and-Processes/`, conforme o caso.

## Sequencia de validacao

1. Executar 2.1 e validar a skill de roteiro.
2. Executar 2.2 e validar a skill de apresentacao HTML.
3. Executar 2.3 para atualizar a skill de plano com criacao de fases apos validacao.
4. Executar 3.1 e parar para validacao do Thiago.
5. Executar 3.2 e parar para validacao do Thiago.
6. Executar 3.3, coletar opiniao do Thiago e so entao criar a v2 final.
7. Repetir a mesma logica para 4.1, 4.2 e 4.3.
8. So depois atualizar skills existentes ou novas na story 4.4.
