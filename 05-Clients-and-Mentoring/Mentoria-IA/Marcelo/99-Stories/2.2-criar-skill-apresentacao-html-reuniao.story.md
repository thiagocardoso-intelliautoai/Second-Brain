# Story 2.2 - Criar skill de apresentacao HTML da reuniao

Status: Draft

## Story

Como Thiago, quero uma skill que transforme um roteiro validado de proxima reuniao em uma apresentacao HTML utilizavel, para que a segunda reuniao ou sessoes futuras tenham material visual claro, pratico e consistente com o objetivo da mentoria.

## Contexto

O arquivo `Apresentacao-Proxima-Reuniao.html` ja existe como apresentacao usada na reuniao de validacao. A nova skill deve partir do roteiro, nao da transcricao bruta, e deve gerar uma apresentacao que ajude a conduzir a sessao, nao uma landing page ou material decorativo.

## Skill proposta

Nome proposto: `criar-apresentacao-html-de-reuniao-de-mentoria`

Local proposto: `C:/Users/thiag/.codex/skills/criar-apresentacao-html-de-reuniao-de-mentoria`

## Acceptance Criteria

1. A skill e criada com `SKILL.md` valido usando o fluxo do `skill-creator`.
2. O input principal e um roteiro validado de reuniao.
3. A skill deixa explicito que nao deve inventar objetivos, materiais ou decisoes ausentes.
4. O HTML gerado deve ser usavel para conduzir a reuniao, com secoes, fluxo, visual limpo e sem excesso de texto.
5. A skill orienta a verificar visualmente o HTML quando houver frontend/HTML local.
6. A skill define quando atualizar apresentacao existente e quando criar nova versao.
7. A skill e validada com `quick_validate.py`.

## Tarefas

- [ ] Ler `skill-creator`.
- [ ] Analisar `Apresentacao-Proxima-Reuniao.html` como referencia de saida, sem copia-la cegamente.
- [ ] Definir estrutura minima de apresentacao: abertura, objetivos, contexto, pratica, decisao e fechamento.
- [ ] Rodar `init_skill.py`.
- [ ] Escrever `SKILL.md` com workflow e criterios de qualidade visual.
- [ ] Validar com `quick_validate.py`.
- [ ] Registrar se a skill deve reutilizar padroes do `html-presentation-designer` quando aplicavel.
