# Output Standards

## Outputs comuns

Thiago costuma produzir:

- PPTX;
- LinkedIn posts;
- n8n automations;
- Claude Skills;
- process documentation;
- internal frameworks;
- client-facing AI/automation recommendations;
- technical handoffs;
- diagnosis documents;
- lead magnets.

## Padrao de entrega

Para qualquer output, sempre separar:

- contexto;
- objetivo;
- inputs;
- decisao tomada;
- output final;
- proximos passos;
- riscos ou pendencias.

## Para documentos estrategicos

Usar estrutura:

1. Problema
2. Consequencia
3. Invalidacao de objecoes
4. Solucao
5. Como funciona na pratica
6. Trade-offs
7. Proximos passos

## Para automacoes

Usar estrutura:

1. Processo atual
2. Dor ou gargalo
3. Tarefa candidata a automacao
4. ROI esperado
5. Modo ideal: assistido, workflow ou agent
6. Ferramentas
7. Pontos de validacao humana
8. Riscos
9. Plano tecnico

## Para LinkedIn

Preservar voz.

Separar:

1. Original
2. Tese
3. Publico
4. Estrutura sugerida
5. Versao sugerida
6. Visual ou formato
7. Risco de soar generico
