# Personal Brand OS

## Posicionamento central

Thiago e o operador-construtor que mostra IA rodando por dentro de uma operacao B2B real.

Nao fala de IA como hype, truque de prompt ou lista de ferramentas. A lente e:

> primeiro o processo, depois o sistema, nunca o truque.

## Lente proprietaria

Built, not prompted.

Em portugues operacional:

- IA boa nao nasce de prompt magico;
- nasce de processo entendido;
- contexto bem organizado;
- criterio humano;
- sistema construido;
- feedback real de operacao.

## Lacuna

Espaco unico:

> O operador construtor que mostra IA rodando por dentro de uma operacao B2B real, nao de fora vendendo consultoria.

## Bandeiras

### Anti-prompt, pro-sistema

Contra a cultura de "um prompt resolve".

### Processo antes de ferramenta

Contra a pressa de colocar agent onde nem o processo esta claro.

Frase-raiz:

> Automacao sem redesenho e bagunca mais rapida.

## Fontes recorrentes de conteudo

1. Skills em producao.
2. Benchmarks reais.
3. Diagnosticos de processo anonimizados.
4. Falhas documentadas.

## Regras de conteudo

1. Sai melhor do que entrou.
2. Destrava decisao, nao execucao.
3. Nao existe em outro lugar.
4. Tem embalagem narrativa.

## Signature visual

Default:

- caderno;
- rascunho;
- notebook raw;
- pensamento sendo construido.

Excecoes:

- Data-Driven para benchmarks;
- Quote Card para frase muito forte;
- Editorial Clean para materia-colab.

## Cuidados

Nao liderar com idade como gimmick.

Nao postar skill em producao com moldura errada. Exemplo: se parecer "automatizei o que cliente paga para consultor fazer", pode gerar leitura ruim. Moldura correta: liberar consultor para pensamento de verdade.

Nao virar agencia vendendo IA. O sujeito do post deve ser a construcao, a falha, o processo ou a decisao.
