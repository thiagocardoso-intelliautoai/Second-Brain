# Pipeline de Conteudo - LinkedIn

## Funcao

Este pipeline transforma ideias brutas em posts sem forcar Thiago a usar um sistema pesado.

Pergunta principal:

> Qual e o menor proximo passo que melhora essa ideia sem apagar a voz dela?

## Etapas

### 1. Captura

Local:

- `Ideias-Brutas/00-Captura-Rapida.md` para despejo rapido;
- `Ideias-Brutas/<slug>/IDEIA.md` quando a ideia ja merece uma unidade propria.

Regra:

- nao polir na captura;
- nao decidir formato cedo demais;
- preservar frase viva.

### 2. Organizacao Da Ideia

Uma ideia organizada deve conter, quando houver:

- original preservado;
- tese;
- erro que combate;
- por que isso importa agora;
- fonte de autoridade;
- possivel formato;
- status.

Nao preencha campo vazio por estetica. Se nao existe, deixe fora.

### 3. Criacao Do Post

Local:

`Posts/<slug>/POST.md`

O post deve ter:

- original preservado ou link para a ideia;
- diagnostico;
- hooks candidatos;
- versao sugerida;
- CTA;
- criterio de qualidade;
- visual sugerido, se fizer sentido.

### 4. Preview

Local:

`Posts/<slug>/PREVIEW.md`

Use para enxergar:

- hook;
- corpo do post;
- CTA;
- sugestao visual;
- status de decisao.

Nao precisa virar app. Markdown bem organizado ja resolve a maior parte.

### 5. Visual

Local:

`Posts/<slug>/visual/brief.md`

Crie apenas quando o post pedir:

- capa;
- carrossel;
- infografico;
- visual de caderno;
- print de autoridade.

## Criterio De Priorizacao

Priorize ideias que:

1. reforcam a lacuna de posicionamento;
2. nascem de operacao real;
3. mostram decisao, erro, processo ou sistema;
4. podem virar post com tensao clara;
5. nao dependem de pesquisa externa longa para existir.

## Tipos De Post

| Tipo | Quando usar |
|---|---|
| Tese contraria | quando a ideia combate hype ou consenso fraco |
| Processo | quando ha etapas replicaveis |
| Falha documentada | quando Thiago tem erro real ou aprendizado vivido |
| Benchmark | quando ha dado ou comparacao externa verificavel |
| Storytelling | quando a historia carrega a tese |
| Materia-colab | quando uma pessoa relevante e o sujeito e a lente de Thiago e o angulo |

## Nao Fazer

- nao criar pauta por pauta;
- nao transformar captura em formulario;
- nao exigir visual para todo post;
- nao medir tudo antes de publicar;
- nao criar automacao antes do fluxo manual estar bom.
