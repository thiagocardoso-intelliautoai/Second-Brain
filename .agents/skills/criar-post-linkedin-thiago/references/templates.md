# Templates

Usar estes templates como estrutura minima. Remover secoes vazias em vez de preencher por estetica.

## `IDEIA.md`

```markdown
---
source: [origem]
created_at: YYYY-MM-DD
status: raw-idea
---

# Ideia - [Titulo]

## Original preservado

[texto original]

## Tese

[ideia central, se clara]

## Erro Que Combate

[erro, hype, confusao ou trade-off]

## Por Que Importa

[por que isso importa agora ou para quem]

## Fonte De Autoridade

[por que Thiago pode falar disso]

## Possivel Formato

[texto / capa / carrossel / video curto / comentario]

## Proximo Passo

[capturar melhor / promover para post / pesquisar claim / aguardar]
```

## `POST.md`

```markdown
---
source: [origem]
derived_from: [caminho da ideia]
created_at: YYYY-MM-DD
updated_at: YYYY-MM-DD
status: draft
do_not_rewrite: true
---

# Post - [Titulo]

## Original preservado

[texto original ou link para IDEIA.md]

## Diagnostico

Forca:

- [o que funciona]

Risco:

- [o que pode dar leitura errada]

Melhor angulo:

> [tese em uma frase]

## Outcome Do Leitor

[o que o leitor deve entender, decidir ou fazer]

## Hooks candidatos

1. [hook]
2. [hook]
3. [hook]

Recomendado: [n]

Motivo:

- [por que esse hook funciona]

## Versao sugerida

[post completo]

## CTA

[CTA especifico]

## Visual sugerido

Formato: [nenhum / capa / carrossel curto]

Estilo: [caderno / data-driven / editorial clean / print de autoridade]

Ideia visual:

- [estrutura visual]

## Status

[draft / pronto para revisao humana / precisa decisao]

## Decisoes Pendentes

- [decisao real, se houver]
```

## `visual/brief.md`

```markdown

## Objetivo

[o que o visual precisa fazer]

[capa unica / carrossel curto]

## Conceito

Titulo:

> [frase visual]

Estrutura:

| Bloco | Conteudo |
|---|---|
| [parte] | [conteudo] |

## Estilo

[caderno / rascunho / data-driven / editorial clean / print de autoridade]

Motivo:

- [por que esse estilo reforca a tese]
```
