## Processo

1. Ler o framework candidato e o contexto onde ele foi usado.
2. Identificar uso real repetido, input recorrente e output observavel.
3. Decidir o menor artefato suficiente: framework, checklist, template, prompt, skill local ou skill global.
4. Evitar skill sem criterio de qualidade ou lugar de teste.
5. Registrar a decisao no backlog quando fizer sentido.
